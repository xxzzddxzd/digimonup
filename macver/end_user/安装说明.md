# PCMacProbe macOS / PlayCover 安装说明

本说明面向只有以下内容的普通用户：

- Apple Silicon Mac；
- 已安装在 `/Applications` 中的 PlayCover；
- 游戏安装包 `1.0.2.ipa`；
- 本插件发布包。

不需要越狱，不需要关闭 SIP，也不需要安装 Theos、Xcode、Python 或其他注入框架。

> `PCMacProbe.dylib` 不能单独拖入 PlayCover。主程序必须包含加载该 dylib 的
> Mach-O 命令，并在修改后重新签名，因此请使用发布包内的安装脚本。

## 兼容范围

- 游戏：`DIGIMON_UP`
- Bundle ID：`jp.co.bandainamcoent.BNEI0442`
- 游戏版本：`1.0.2 (38)`
- 架构：Apple Silicon / arm64
- 运行方式：PlayCover

Unity 函数偏移与游戏版本绑定。不要把这份 dylib 注入其他版本或其他游戏。

## 发布包内容

解压后应当看到：

```text
PCMacProbe-mac-1.0.2/
├── PCMacProbe.dylib
├── pc_macho_inject
├── 安装插件.command
├── 安装说明.md
└── THIRD_PARTY_NOTICES.md
```

请保持这些文件位于同一个文件夹内。

## 安装步骤

1. 打开 PlayCover。
2. 双击 `1.0.2.ipa`，或将它拖入 PlayCover，完成游戏安装。
3. 确认 `DIGIMON_UP` 已出现在 PlayCover 资料库中。第一次启动不是必需步骤。
4. 如果游戏正在运行，先正常退出游戏。
5. 解压插件发布包。
6. 右键点击 `安装插件.command`，选择“打开”，再确认运行。
7. 终端显示“安装完成”后，回到 PlayCover 启动游戏。

进入游戏后，屏幕边缘会出现可拖动的“加速”悬浮按钮。点击按钮可打开面板，
使用滑块设置 1× 到 10× 的整数倍速。选择值会自动保存，其他插件功能默认启用。

## macOS 阻止脚本时

不要全局关闭 Gatekeeper。打开“终端”，输入 `cd` 和一个空格，把解压后的插件
文件夹拖入终端窗口，然后按回车。接着依次执行：

```sh
xattr -dr com.apple.quarantine .
chmod +x 安装插件.command pc_macho_inject
./安装插件.command
```

## 更新或重新安装游戏

PlayCover 更新、重新安装或重新转换 IPA 后，游戏主程序会被替换，插件加载命令也会
消失。退出游戏并再次运行 `安装插件.command` 即可。安装脚本可以重复运行。

每次运行脚本都会在 PlayCover 的 `Applications` 目录旁创建带时间戳的备份，例如：

```text
jp.co.bandainamcoent.BNEI0442.app.PCMacProbeBackup.20260716_120000
```

## 卸载插件

最简单可靠的方式是在 PlayCover 中删除游戏，然后用原始 `1.0.2.ipa` 重新安装。这样会
恢复没有 dylib 加载命令的主程序。是否保留游戏数据请按 PlayCover 的提示自行选择。

不要只删除 `Frameworks/PCMacProbe.dylib`：主程序仍会尝试加载它，游戏将无法启动。

## 常见问题

### 游戏能够启动，但看不到“加速”按钮

确认游戏版本是 `1.0.2 (38)`，完全退出游戏后重新运行安装脚本，再从 PlayCover 启动。

### 提示没有安装 DIGIMON_UP

需要先由 PlayCover 安装 `1.0.2.ipa`。脚本默认查找：

```text
~/Library/Containers/io.playcover.PlayCover/Applications/
jp.co.bandainamcoent.BNEI0442.app
```

### 更新后游戏无法启动

新版游戏的 UnityFramework 很可能已经改变。请不要继续使用 1.0.2 的 dylib；先在
PlayCover 中重新安装未修改的 IPA，等待对应游戏版本的插件更新。

### 画面方向或尺寸不正确

这是 PlayCover 的单独显示设置，与 dylib 注入无关。在游戏的 PlayCover 设置中选择
自定义分辨率和竖屏方向，然后重新启动游戏。
